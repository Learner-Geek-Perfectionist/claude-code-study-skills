# Study-Master Skill 设计文档

**日期**：2026-03-12
**作者**：Claude
**版本**：1.0

## 1. 概述

### 1.1 目标

创建一个名为 `study-master` 的 skill，帮助程序员深度学习：
- 开源项目源码
- 协议栈实现（如 TCP/IP、HTTP）
- 特定编程语言或框架的内部机制

面向新手学习者，提供从 0 到 1 的完整学习路径。

### 1.2 核心特性

- **深度解析型文档**：每个主题生成 50-100 页的详细学习材料
- **混合组织方式**：快速导览（场景驱动）+ 模块深度解析
- **多层次展示**：调用关系图 → 伪代码 → 真实代码 → 实现细节
- **教科书风格**：结构化、渐进式、有逻辑关联
- **实时进度反馈**：生成过程中显示当前进度
- **一次性生成**：自动生成完整的学习文档系列

### 1.3 使用方式

```bash
/study-master <topic>
/study-master <topic> --source <path>
```

## 2. 整体架构

### 2.1 目录结构

生成的文档存放在：

```
<topic>/
├── 00-overview.md          # 快速导览（场景驱动）
├── 01-module-xxx.md        # 模块1深度解析
├── 02-module-yyy.md        # 模块2深度解析
├── ...
├── appendix-references.md  # 参考资料和索引
└── .study-meta.json        # 元数据（生成时间、版本等）
```

### 2.2 Skill 文件结构

`SKILL.md` 包含：
- Frontmatter（name、description）
- 参数解析逻辑
- 8 阶段工作流程定义
- 文档生成规范引用
- 质量控制检查点

## 3. 工作流程

### 3.1 核心流程（8 个阶段任务）

skill 会创建并按顺序执行以下任务：

**阶段 1：主题识别与准备**
- 解析 `<topic>` 参数，判断类型（项目/协议/语言内部机制）
- 确定源码位置（本地路径、GitHub URL、或协议规范文档）
- 创建目标目录 `<topic>/`
- 输出：`正在准备学习主题: <topic>...`

**阶段 2：项目结构分析**
- 扫描目录结构、关键文件
- 识别主要模块和依赖关系
- 生成项目架构概览
- 输出：`正在分析项目结构...`

**阶段 3：生成学习大纲**
- 基于分析结果，确定章节划分
- 识别核心模块和学习路径
- 生成文档目录结构
- 输出：`已生成学习大纲，共 X 个章节`

**阶段 4：生成快速导览**
- 编写 `00-overview.md`
- 包含：项目简介、核心概念、典型场景的完整流程
- 输出：`正在生成快速导览章节...`

**阶段 5-N：逐模块深度解析**
- 按大纲顺序，逐个生成模块文档
- 每个模块包含：调用关系图 → 数据结构 → 核心算法 → 代码实现
- 输出：`正在解析模块 [X/N]: <module-name>...`

**最后阶段：整合与收尾**
- 生成参考资料索引
- 创建 `.study-meta.json` 元数据
- 输出：`学习文档生成完成！位置: <topic>/`

### 3.2 分析策略

**开源项目**：
- 使用 Glob/Grep 扫描源码结构
- 使用 LSP（如果可用）分析函数调用关系
- 识别入口函数和核心数据流

**协议规范**：
- 读取 RFC 文档或规范文件
- 提取关键章节和协议状态机
- 结合实现代码（如果提供）

**语言内部机制**：
- 结合官方文档和实现源码
- 重点分析运行时机制和内存模型

## 4. 文档内容结构

### 4.1 快速导览章节（00-overview.md）

包含以下部分：

1. **项目/协议简介**（1-2页）
   - 背景、目标、应用场景

2. **核心概念速览**（2-3页）
   - 关键术语、设计理念

3. **典型场景剖析**（5-10页）
   - 选择 1-2 个典型场景
   - 追踪完整执行流程
   - 串联各个模块

4. **架构全景图**（1页）
   - 模块关系图和数据流向
   - 使用 Mermaid 图表

5. **学习路线图**（1页）
   - 后续章节导航
   - 推荐学习顺序

### 4.2 模块深度解析章节（01-module-xxx.md）

每个模块文档采用统一结构：

**1. 模块概述**（1-2页）
- 模块职责和边界
- 在整体架构中的位置
- 与其他模块的交互

**2. 多层次代码展示**（10-15页）
- **第一层：调用关系图** - 使用 Mermaid 图展示函数调用链
- **第二层：伪代码** - 用简化的伪代码描述核心逻辑
- **第三层：真实代码** - 展示关键代码片段，逐段解析
- **第四层：实现细节** - 边界处理、性能优化、设计权衡

**3. 数据结构深度解析**（3-5页）
- 核心数据结构定义
- 字段含义和设计考量
- 内存布局和性能影响

**4. 关键算法剖析**（5-8页）
- 算法原理和时间复杂度
- 代码实现逐行解释
- 边界情况和异常处理

**5. 设计决策分析**（2-3页）
- 为什么这样设计？
- 有哪些替代方案？
- 权衡取舍是什么？

## 5. 技术实现细节

### 5.1 文档生成规范

skill 将严格遵守 `~/.claude/skills/_shared/format-rules.md` 中的所有规范：

**链接格式**：
- 源码位置：`> 📍 源码：[文件名:起始行-结束行](相对路径#L起始行)`
- 文档交叉引用：`[术语名](./目标文件.md#锚点)`
- 所有源码文件名必须是超链接

**图表格式**：
- 强制使用 Mermaid，禁止 ASCII art
- 节点文本 ≤ 25 字符
- 同层节点 ≤ 6 个，总节点 ≤ 20 个

**表格规范**：
- 表格上方添加图注：`> **图 N** 标题——描述`
- 字段详解表格中的结构体引用必须加超链接

**反引号规则**：
- 纯文本标识符用反引号
- 超链接标识符不加反引号

**代码块**：
- 必须指定语言标识（c/bash/mermaid/text）

**数学表达式**：
- 使用 LaTeX 格式（`$...$` 或 `$$...$$`）
- 禁止 Unicode 数学符号

### 5.2 关键工具使用

- **Glob/Grep**：扫描和搜索源码
- **Read**：读取源文件和文档
- **Write**：生成 markdown 文档
- **LSP**（如果可用）：分析函数调用关系
- **TaskCreate/TaskUpdate**：进度管理和反馈

### 5.3 质量控制

**文档质量检查**：
- 每个章节长度控制（模块章节 15-25 页）
- 代码示例完整性检查
- 逻辑连贯性验证
- 生成后逐文件用 Read 检查乱码
- 发现 U+FFFD 或格式问题立即修正

**内容完整性**：
- 确保每个模块都有调用关系图
- 验证所有源码引用都有正确的链接
- 检查交叉引用的有效性

### 5.4 错误处理

**主题不存在或路径无效**：
- 提示用户提供正确路径
- 建议可能的主题名称

**源码过于复杂**：
- 提示可能需要较长时间
- 询问是否继续

**生成中断**：
- 保存已生成的部分
- 支持断点续传（通过检查已存在的文件）

## 6. 质量检查 Hook

### 6.1 Hook 设计

**名称**：`format-validator`

**触发时机**：当 study-master skill 生成文档后，自动触发检查

**检查项**：

1. **链接格式检查**
   - 检测 `file:///` 绝对路径
   - 检测链接文本中的反引号（`` [`xxx`]() ``）
   - 验证源码位置格式是否符合规范

2. **图表格式检查**
   - 检测 ASCII art 字符（`┌──┐`、`├──`、`└──` 等）
   - 检查 Mermaid 图表节点数量（同层 ≤ 6，总数 ≤ 20）

3. **反引号规则检查**
   - 检测超链接内的反引号（`` [`xxx`](url) ``）

4. **数学符号检查**
   - 检测 Unicode 数学符号（²³⁰、×、≥、≤ 等）
   - 提示应使用 LaTeX 格式

5. **代码块检查**
   - 检测无语言标识的代码块

6. **乱码检查**
   - 检测 U+FFFD 替换字符

**输出方式**：
- 如果发现违规，输出详细报告（文件名、行号、违规类型）
- 如果全部通过，输出 `✅ 格式检查通过`

### 6.2 Hook 实现方式

使用 shell 脚本实现，接收文档目录作为参数：
- 遍历所有 `.md` 文件
- 使用正则表达式检测违规模式
- 输出结构化的检查报告
