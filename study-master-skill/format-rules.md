# 文档格式规范

所有 study-master 生成的 Markdown 文档必须遵守以下规范。

## 链接格式

**核心原则：所有源码标识符（文件名、结构体、函数）必须可点击，禁止纯文本。**

- 必须使用相对路径（前缀为 `source_path_prefix`），禁止 `file:///` 绝对路径
- 链接文本不加反引号（反引号破坏超链接样式）
- **源码位置格式**：`> 📍 源码：[文件名:起始行-结束行](相对路径#L起始行)`
  - 源码位置标注中，表格内只写文件名（避免表格过宽）
  - 但函数名/结构体名在表格中出现时仍然必须有超链接，不受此简化规则影响
- **文档间交叉链接**：`[术语名](./目标文件.md#锚点)`
  - 锚点遵循 GFM 规则：全小写，空格替换为 `-`，去除特殊字符
  - **跨文件引用的锚点必须纯 ASCII**（VS Code 对含中文的跨文件锚点导航有 bug）：
    - 结构体/函数标题直接用英文：`### struct ddsi_xxx`、`### ddsi_xxx_func`
    - 中文小节标题如果可能被其他章节引用，必须添加自定义锚点：`### 内存布局 {#memory-layout}`
    - 页内链接（同一文件内）可以使用中文锚点

## 图表格式

- **所有图表必须使用 Mermaid**，禁止 ASCII art / box-drawing 字符（`┌─┐│└┘├┤` 等）
- **内存布局例外**：嵌套包含关系（如内存池→缓冲区→消息→片段）Mermaid 渲染效果差，改为生成独立 `.svg` 文件到 `study/<topic>/` 目录，在 Markdown 中用 `![描述](文件名.svg)` 引用。SVG 用 `<rect>` 嵌套表达包含层次，用 `<text>` 标注结构体名，配色使用暗色主题友好的低饱和色（深色背景 + 亮色边框/文字）。禁止内嵌 SVG（渲染器会过滤内容）。
- **节点文本 ≤ 25 字符**，超长用 `<br/>` 换行
- **图表密度限制**：同一层级节点数 ≤ 6，节点总数 ≤ 20，超过时拆分为多张图表

## 反引号规则

- 有超链接 → 不加反引号；无超链接的标识符 → 必须加反引号

## 文本编码

- UTF-8 编码，禁止 box-drawing Unicode 字符（`│`, `├`, `└`, `─` 等）

## 代码块

- 所有代码块必须指定语言标识（C: ` ```c `，Shell: ` ```bash `，Mermaid: ` ```mermaid `）
- **伪代码使用最接近的语言标识**（如 C 风格伪代码用 ` ```c `），禁止用 ` ```text ` 包裹有编程语言语法特征的代码
- 禁止无语言标识的裸代码块

## 数学表达式

- **所有数学符号必须使用 LaTeX 格式**（GitHub Markdown 原生支持）
- **行内公式**：用 `$...$` 包裹，如 `$2^{31}$`、`$N \times 2^{20}$`
- **块级公式**：用 `$$...$$` 包裹，独立成段
- **禁止使用 Unicode 数学符号**：
  - ❌ 上标字符：²³⁰¹⁴⁵⁶⁷⁸⁹
  - ❌ 运算符号：×（乘号）、÷（除号）
  - ❌ 比较符号：≥、≤、≠、≈
  - ✅ 统一用 LaTeX：`$2^{31}$`、`$\times$`、`$\geq$`、`$\leq$`、`$\neq$`

**常用 LaTeX 符号对照**：

| 含义 | Unicode | LaTeX |
|------|---------|-------|
| 幂运算 | 2³¹ | `$2^{31}$` |
| 乘法 | N × M | `$N \times M$` |
| 大于等于 | ≥ | `$\geq$` |
| 小于等于 | ≤ | `$\leq$` |
| 不等于 | ≠ | `$\neq$` |
| 约等于 | ≈ | `$\approx$` |
| 分数 | 1/2 | `$\frac{1}{2}$` |
