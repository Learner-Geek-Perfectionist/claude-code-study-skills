# Study-Master Skill

## 概述

帮助程序员深度学习：
- 开源项目源码
- 协议栈实现（如 TCP/IP、HTTP）
- 特定编程语言或框架的内部机制

生成 50-100 页的深度解析型学习文档，采用教科书风格。

## 使用方式

```bash
/study-master <topic>
/study-master <topic> --source <path>
```

## 参数说明

- `<topic>`：学习主题名称（必需）
- `--source <path>`：源码或文档路径（可选）

## 工作流程

skill 会创建并按顺序执行以下 8 个阶段任务：

### 阶段 1：主题识别与准备

- 解析 `<topic>` 参数，判断类型（项目/协议/语言内部机制）
- 确定源码位置（本地路径、GitHub URL、或协议规范文档）
- 创建目标目录 `<topic>/`
- 输出进度：`正在准备学习主题: <topic>...`

### 阶段 2：项目结构分析

- 使用 Glob/Grep 扫描目录结构、关键文件
- 识别主要模块和依赖关系
- 生成项目架构概览
- 输出进度：`正在分析项目结构...`

### 阶段 3：生成学习大纲

- 基于分析结果，确定章节划分
- 识别核心模块和学习路径
- 生成文档目录结构
- 输出进度：`已生成学习大纲，共 X 个章节`

### 阶段 4：生成快速导览

- 编写 `00-overview.md`
- 包含：项目简介、核心概念、典型场景的完整流程
- 输出进度：`正在生成快速导览章节...`

### 阶段 5-N：逐模块深度解析

- 按大纲顺序，逐个生成模块文档
- 每个模块包含：调用关系图 → 数据结构 → 核心算法 → 代码实现
- 输出进度：`正在解析模块 [X/N]: <module-name>...`

### 最后阶段：整合与收尾

- 生成参考资料索引 `appendix-references.md`
- 创建 `.study-meta.json` 元数据
- 输出进度：`学习文档生成完成！位置: <topic>/`

## 文档结构

### 快速导览章节（00-overview.md）

1. 项目/协议简介（1-2页）
2. 核心概念速览（2-3页）
3. 典型场景剖析（5-10页）- 追踪完整执行流程
4. 架构全景图（1页）- 使用 Mermaid 图表
5. 学习路线图（1页）

### 模块深度解析章节（01-module-xxx.md）

每个模块文档包含：

1. **模块概述**（1-2页）- 职责、边界、交互关系
2. **多层次代码展示**（10-15页）
   - 第一层：调用关系图（Mermaid）
   - 第二层：伪代码
   - 第三层：真实代码片段
   - 第四层：实现细节
3. **数据结构深度解析**（3-5页）
4. **关键算法剖析**（5-8页）
5. **设计决策分析**（2-3页）

## 文档生成规范

**严格遵守** `~/.claude/skills/_shared/format-rules.md` 中的所有规范：

- 源码位置：`> 📍 源码：[文件名:起始行-结束行](相对路径#L起始行)`
- 强制使用 Mermaid，禁止 ASCII art
- 节点文本 ≤ 25 字符，同层节点 ≤ 6 个，总节点 ≤ 20 个
- 表格上方添加图注：`> **图 N** 标题——描述`
- 纯文本标识符用反引号，超链接标识符不加反引号
- 代码块必须指定语言标识（c/bash/mermaid/text）
- 数学表达式使用 LaTeX 格式（`$...$`），禁止 Unicode 数学符号

## 质量控制

- 每个模块章节 15-25 页
- 生成后逐文件用 Read 检查乱码
- 发现 U+FFFD 或格式问题立即修正
- 确保所有源码引用都有正确的超链接
